# Skill: Testing the Sibme Copilot Module

## When to use this skill
Read this file before any test cycle or test case writing session that
involves the Sibme Copilot module. Read it in addition to `test_cycle.md`,
not instead of it.

## Load business rules first
Always call `get_docs("copilot")` before planning any Copilot test steps.
The doc `context_server/docs/copilot.md` is the authoritative source for
Copilot-specific rules. If that file does not exist yet, flag it as a
documentation gap in the report.

## Non-determinism rules
Copilot responses are generated by an AI and are non-deterministic.

| What to assert | What NOT to assert |
|---|---|
| A response EXISTS | The exact wording of the response |
| The response is non-empty (len > 0) | Specific facts or figures in the response |
| The response is relevant to the prompt topic | The response matches a stored snapshot |

## Timing
- Wait up to **60 seconds** for a Copilot response before marking the step FAIL.
- Do not use the default Playwright timeout for Copilot interactions — set it explicitly.
- If the response arrives after 45 seconds, log a WARN with the actual elapsed time.

## Classifying bad responses
| Observation | Status | Action |
|---|---|---|
| No response after 60 s | FAIL | Screenshot, note timeout |
| Empty string returned | FAIL | Screenshot, note empty response |
| Response is off-topic or hallucinates | WARN | Note content, do not FAIL |
| Response is in wrong language | WARN | Note locale mismatch |
| Response is relevant and non-empty | PASS | No screenshot needed |

## Account requirement
Copilot is only available to users in accounts with the AI add-on enabled.
Verify `TARGET_ACCOUNT` in `.env` has this add-on before running Copilot tests.
If the Copilot module is not visible after login, log WARN "AI add-on not enabled
for this account" and skip the cycle.
